## 3. Release signing (ci-signing)

- [x]3.1 Add an ed25519 signing private key as a CI secret (never committed, never logged)
- [x]3.2 On release/tag, build the daemon release-only and sign the OTA release manifest
      (`build/release-manifest.json`, generated by `make release-manifest` owned by the `ota`
      change) with the CI-secret key; use `trap` to ensure the temp keyfile is removed on failure
- [x]3.3 Verify the signed output with the committed public key as a pipeline check
- [x]3.4 Ensure non-release push/PR builds do not require the signing key
